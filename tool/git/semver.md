# Semantic versioning

Semantic versioning of a software project.

Pattern: `MAJOR.MINOR.PATCH`

Example: `v1.2.3`:
1. `MAJOR` version: increments when commiting **incompatible** API changes
1. `MINOR` version: increments when commiting **backwards-compatible functionalities**
1. `PATCH` version: increments when commiting **backwards-compatible bug fixes**
