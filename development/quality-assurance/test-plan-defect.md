# Software testing - test plan defect

When a test case fails, this causes one or more test defects

A defect can be summarized like this:
- summary
- description
- reproduction steps
- expected behavior
- observed behavior
- impact: what test cases are blocked by the defect
- severity
- workaround
- notes

A defect severity can be graduated like this:
- blocker: system cannot be released
- critical: severely impacts the system
- major: relatively severe problem
- normal: inconvenience but there's a workaround
- minor: may not be noticed
- trivial: should not even be noticed (eg typo)
- enhancement
