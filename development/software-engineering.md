# software engineering

goal: produce a efficient and reliable software

uses well-defined scientific principles, methods and procedures

software engineering includes:
- requirements analysis
- system analysis
- system design
- code design
- testing
- deployment
- maintenance
- updates
- release mgmt

## updates and maintenance

software evolves in a cycle that typically is:
- change request
- impact analysis
- release planning
- system update
- system release

## software evolution categories

static type (s-type): works strictly according to defined specs and solutions, eg: calculator

practical type (p-type): specs can be described but solution isn't obvious, eg: game

e-type (embedded): works as the requirements of the reals world changes, eg: taxes software
