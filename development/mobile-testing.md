# mobile testing

## testing devices

- a greater number of mobile devices than desktop devices
- a greater number of OS
- this leads to an increase in the number of tests to be done
- when timing requires it, focus mobile testing on critical parts of the software (UI/UX)
- prioritize devices and OS according to the market and your users

## carriers and in-the-wild testing problems

- there's no way to replicate real-world connection issues in diverse locations
- test gps-based intelligence is complicated
- test ip-based localization is complicated
- when testing the app, try to know where the information is sent, is it the most appropriate node?
